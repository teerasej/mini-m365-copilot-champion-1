# Exercise: Copilot Chat and Work IQ

## Exercise Overview

- **เวลา:** 09:00–10:30 (90 นาที)
- **เป้าหมาย:** ใช้ Copilot Chat กับบริบทงานเพื่อร่าง follow-up สรุปสัปดาห์ และเตรียมประชุม
- **ผลลัพธ์:** email draft, weekly summary และ meeting brief ที่ตรวจสอบแล้ว

## Prerequisites

- บัญชี Microsoft 365 Copilot ที่เข้าถึงข้อมูลการทำงานของผู้เรียนได้ตามสิทธิ์
- เลือกข้อมูลจริงที่ไม่เป็นความลับ หรือใช้สถานการณ์ตัวอย่างด้านล่าง
- ห้ามวางข้อมูลส่วนบุคคล ข้อมูลลับ หรือไฟล์ที่ไม่ได้รับอนุญาตลงใน prompt

> **หมายเหตุเกี่ยวกับภาพ:** ภาพมาจาก source exercise เดิมโดยไม่แก้ไข จึงอาจเห็นชื่อผู้ใช้ tenant บริษัท หรือ prompt เดิม ให้ยึดชื่อ ค่า และ prompt ที่เขียนในขั้นตอนปัจจุบัน

## Scenario 1: ติดตามข้อตกลงกับผู้ขาย

พวกเรารอข้อมูลราคาและกำหนดส่งจากผู้ขาย ต้องส่งอีเมลสุภาพที่บอกสิ่งที่ค้างและวันตอบกลับ โดยยังไม่ถือว่าข้อตกลงได้รับอนุมัติ

### Practice 1: ร่าง follow-up email

#### Steps

1. เปิด [Microsoft 365 Copilot](https://m365copilot.com/) ด้วยบัญชีที่ trainer กำหนด

![หน้าเริ่มต้น Microsoft 365 Copilot Chat](images/open-microsoft-365-copilot-chat.png)

2. เลือก **Chat** แล้วเลือก work context ที่ tenant อนุญาต หาก UI แสดงตัวเลือก **Work** และ **Web** ให้เลือก **Work** สำหรับกิจกรรมนี้

![เลือก Work context ใน Copilot Chat](images/select-work-context.png)

3. วาง prompt นี้และแทนข้อมูลในวงเล็บเหลี่ยม:

```text
ช่วยร่างอีเมลติดตามผู้ขายเป็นภาษาไทย
เป้าหมาย: ขอราคาอัปเดต กำหนดส่ง และเงื่อนไขการสนับสนุน
บริบท: เราคุยกันล่าสุดวันที่ [วันที่] และต้องใช้ข้อมูลเพื่อเตรียมการพิจารณาภายใน
รูปแบบ: หัวเรื่อง + เนื้อหาไม่เกิน 140 คำ + bullet รายการที่ขอ
โทน: สุภาพ กระชับ ไม่กล่าวว่าข้อตกลงได้รับอนุมัติแล้ว
ก่อนจบ ให้ระบุข้อมูลที่ฉันควรตรวจสอบเอง
```

4. ตรวจชื่อ วันที่ รายการที่ขอ และคำกล่าวอ้าง แล้วแก้ส่วนที่ AI เดา
5. ให้คู่ของพวกเราอ่านเฉพาะร่างและชี้หนึ่งจุดที่อาจทำให้ผู้รับเข้าใจผิด

## Scenario 2: สรุปสัปดาห์และเตรียมประชุม

พวกเราต้องกลับเข้าสู่จังหวะงานหลังมีอีเมล เอกสาร และการประชุมหลายรายการ เป้าหมายคือสร้าง briefing ไม่ใช่ให้ AI ตัดสินลำดับความสำคัญแทนพวกเรา

### Practice 2: สร้าง weekly summary และ meeting brief

#### Steps

1. ใช้ prompt สรุปงาน โดยจำกัดช่วงเวลาและขอ citation/link กลับไปยังแหล่งข้อมูลเมื่อระบบรองรับ:

```text
สรุปงานของฉันในช่วง [วันเริ่ม] ถึง [วันสิ้นสุด] จากข้อมูลที่ฉันมีสิทธิ์เข้าถึง
จัดเป็น 4 หัวข้อ: Completed, In progress, Waiting for others, Risks or decisions
สำหรับแต่ละข้อ ระบุแหล่งที่มาและวันที่ถ้ามี
หากข้อมูลไม่พอ ให้เขียนว่า "ไม่พบข้อมูล" และอย่าคาดเดา
ปิดท้ายด้วย 3 รายการที่ฉันควรตรวจสอบเอง
```

2. เปิดแหล่งอ้างอิงอย่างน้อย 2 รายการและตรวจว่าข้อสรุปตรงกับต้นฉบับ
3. เลือก meeting ที่จะเกิดขึ้นและใช้ prompt นี้:

```text
ช่วยเตรียม meeting brief สำหรับการประชุม [ชื่อ/หัวข้อ] วันที่ [วันเวลา]
สรุป: เป้าหมาย ผู้เข้าร่วมที่ทราบ บริบทล่าสุด ประเด็นค้าง การตัดสินใจที่ต้องการ และคำถาม 5 ข้อ
แยก Fact ออกจาก Assumption และใส่แหล่งที่มาสำหรับ Fact
หากพวกเราเข้าถึงข้อมูลใดไม่ได้ ให้ระบุข้อจำกัดตรง ๆ
```

4. ขีดเส้นใต้ 1 ข้อที่ Copilot ช่วยประหยัดเวลาและวง 1 ข้อที่ยังต้องใช้ judgment ของพวกเรา

## Checkpoint

- ทุก draft แยก fact, assumption และข้อมูลที่ยังไม่พบ
- เปิดตรวจแหล่งอ้างอิงอย่างน้อย 2 จุดก่อนนำไปใช้
- ไม่ส่งอีเมลจากแบบฝึกหัดนี้; ใช้เป็นร่างเพื่อ review เท่านั้น

## Expected Output

ชุดข้อความ 3 ชิ้น: vendor follow-up, weekly-work summary และ meeting brief พร้อมร่องรอยการตรวจสอบ

## Optional Extension

เลือกหนึ่ง prompt แล้วขอ Copilot แสดงข้อมูลเดิมในรูปแบบใหม่ เช่น ตาราง 3 คอลัมน์ จากนั้นเปรียบเทียบว่ารูปแบบใดช่วยให้ตัดสินใจได้ง่ายกว่า
