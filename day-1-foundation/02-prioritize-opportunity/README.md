# Exercise: Prioritize the Pain Point

## Exercise Overview

- **เป้าหมาย:** เลือก pain point ที่คุ้มค่าจะนำไปออกแบบ workflow ใหม่
- **ผลลัพธ์:** pain-point theme ที่เลือก พร้อมคะแนน ผลโหวต และหลักฐานสนับสนุน

## Prerequisites

- Work Friction Map จาก Section 1
- post-it ปากกา dot sticker คนละ 3 จุด flipchart และ [Pain Point Priority Matrix](../../templates/pain-point-priority-matrix.md)

## Scenario 1: เลือก paint point ที่คุ้มค่าลงแรง

ทีมไม่มีเวลาปรับ หรือแก้ทุกปัญหาพร้อมกัน จึงต้องรวมปัญหาที่คล้ายกันและเลือกที่พบบ่อยที่สุด และสร้างความลำบาก และมีผลต่องานจริง

### Practice 1: จัดกลุ่ม โหวต และเลือก pain point

#### Steps

1. แต่ละคนเขียนความลำบากจากงานจริง 3 เรื่อง เรื่องละหนึ่ง post-it **หรือถ้ามี pain point ที่เขียนไว้แล้วให้เริ่มจากจุดนั้นได้**
2. ให้คะแนนแต่ละ theme ใน [Pain Point Priority Matrix](../../templates/pain-point-priority-matrix.md) จาก 1–3 ด้าน business impact, frequency, frustration และ feasibility
3. ให้สมาชิกโหวตคะแนน คนละ 3 จุด โหวตได้ทั้งหมดในเรื่องเดียวหรือแบ่งหลายเรื่อง
4. เลือกโหวต pain point
5. เลือก theme ที่ได้คะแนนสูง มีผลโหวตสนับสนุน และทีมมีอำนาจทดลองได้
6. บันทึกตัวอย่างหรือ note สั้น ๆ ที่อธิบายว่า pain point นี้เกิดขึ้นบ่อยและกระทบงานอย่างไร

## Checkpoint

- เลือก pain-point theme เดียวที่มีคะแนน ผลโหวต และหลักฐานสนับสนุน
- scope เล็กพอสำหรับการออกแบบหรือทดลองหนึ่ง workflow
- ยังไม่ล็อกคำตอบว่า “ต้องใช้ AI” ก่อนออกแบบงาน

## Expected Output

Pain-point theme ที่ทีมเลือก พร้อมคะแนน ผลโหวต และเหตุผลว่าทำไมทีมเลือกเรื่องนี้
