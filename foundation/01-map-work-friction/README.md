# Exercise: Map Work Friction

## Exercise Overview

- **เวลา:** 09:00–10:30 (90 นาที)
- **เป้าหมาย:** เห็นภาพงานประจำวันและระบุจุดที่ช้า ซ้ำ เครียด หรือไม่ชัด
- **ผลลัพธ์:** Work Friction Map หนึ่งแผ่นและ pain point ที่วงไว้ 3 จุด

## Prerequisites

- post-it 10–15 แผ่นต่อคน ปากกาเมจิก flipchart และ [Work Friction Map](../../templates/work-friction-map.md)
- ปิดหน้าจอและเก็บโทรศัพท์ กิจกรรมนี้ไม่ใช้แอป

## Scenario 1: มองงานเหมือนครัวที่กำลังยุ่ง

ทีมของคุณมีงานเข้ามาหลายทาง บางงานรอข้อมูล บางงานต้องทำซ้ำ และบางงานไม่ชัดว่าใครรับช่วงต่อ ก่อนเลือก AI คุณต้องเห็น bottleneck ให้ชัด เหมือนหาจุดที่อาหารค้างอยู่ในครัวก่อนซื้ออุปกรณ์ใหม่

### Practice 1: เขียนแผนที่งานประจำวัน

#### Steps

1. เขียนงานหนึ่งอย่างต่อ post-it หนึ่งแผ่น เริ่มทุกใบด้วยคำกริยา เช่น “ตรวจคำขอ” หรือ “สรุปการประชุม”
2. วาง post-it เป็น 3 กลุ่ม: **Individual**, **Team**, และ **Organization**
3. เพิ่มเครื่องหมาย `S` สำหรับงานที่ช้า, `R` สำหรับงานซ้ำ, `F` สำหรับงานที่สร้างความกังวล และ `?` สำหรับงานที่ไม่ชัด
4. เลือกงานหนึ่งใบแล้วอธิบายให้คู่ของคุณฟังว่า “ฉันอยากให้งานนี้ง่ายขึ้น เร็วขึ้น หรือชัดขึ้นอย่างไร”
5. ย้ายงานทั้งหมดลง [Work Friction Map](../../templates/work-friction-map.md)
6. วง pain point ที่เด่นที่สุด 3 จุด โดยยังไม่เสนอเครื่องมือหรือ solution

## Checkpoint

- มีงานอย่างน้อย 8 งาน ครบมากกว่า 1 ระดับ
- pain point ทุกจุดบอกได้ว่าเกิดตรงไหนและสร้างผลกระทบอะไร
- หากคำอธิบายเป็นเพียง “งานเยอะ” ให้ถามต่อว่า “งานส่วนใด รออะไร และใครได้รับผล”

## Expected Output

Work Friction Map ที่แยกงานตามระดับและชี้ 3 จุดติดขัดด้วยหลักฐานจากประสบการณ์จริง

## Optional Extension

ใช้ post-it สีอื่นเขียนสิ่งที่ AI “อาจช่วย” และสิ่งที่ “ยังไม่แน่ใจ” อย่างละ 1 ใบ แล้ววางไว้ข้าง pain point โดยยังไม่ตัดสินใจเลือกใช้
