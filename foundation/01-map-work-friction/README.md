# Exercise: Map Work Friction

## Exercise Overview

- **เวลา:** 09:00–10:30 (90 นาที)
- **เป้าหมาย:** ให้เราสามารถสังเกตเห็นภาพงานประจำวันและระบุจุดที่ช้า ซ้ำ เครียด 
- **ผลลัพธ์:** Work Friction Map หนึ่งแผ่นและ pain point ที่วงไว้ 3 จุด

## Prerequisites

- post-it ต่อคน ปากกาเมจิก flipchart และ [Work Friction Map](../../templates/work-friction-map.md)

## Scenario 1: มองงานเหมือนครัวที่กำลังยุ่ง

ทีมของพวกเรามีงานเข้ามาหลายทาง บางงานรอข้อมูล บางงานต้องทำซ้ำ และบางงานไม่ชัดว่าใครรับช่วงต่อ แม้แต่ตอนทำงานใน nextflow ก่อนเลือก AI พวกเราต้องเห็น bottleneck ให้ชัด เหมือนหาจุดที่อาหารค้างอยู่ในครัวก่อนซื้ออุปกรณ์ใหม่

### Practice 1: เขียนแผนที่งานประจำวัน

#### ขั้นตอนการทำกิจกรรม

1. เขียนงานหนึ่งอย่างต่อ post-it หนึ่งแผ่น เริ่มทุกใบด้วยคำกริยาในการทำงานนั้นๆ เช่น “ตรวจคำขอ” หรือ “สรุปการประชุม”
2. วาง post-it เป็น 3 ระดับกลุ่ม:
	- **Individual**: งานที่ทำคนเดียวและควบคุมได้เอง
	- **Team**: งานที่ต้องประสานหรือส่งต่องานในทีม
	- **Organization**: งานที่เกี่ยวข้องข้ามทีม นโยบาย ระบบส่วนกลาง หรือขั้นตอนระดับองค์กร
3. เขียนเพิ่มเครื่องหมายกำกับงาน:
	- `S`: งานที่ช้า (slow)
	- `R`: งานที่ทำซ้ำ (repetitive)
	- `F`: งานที่สร้างความกังวล เครียด หมดไฟ (frustrating)
	- `?`: งานที่มีข้อมูลไม่ชัดเจน จนไม่สามารถระบุปัญหาได้ 
4. เลือกงานหนึ่งใบแล้วอธิบายให้คนในกลุ่มของพวกเราฟังว่า “ฉันอยากให้งานนี้ง่ายขึ้น เร็วขึ้น หรือชัดขึ้นอย่างไร”
5. ย้ายงานทั้งหมดลง [Work Friction Map](../../templates/work-friction-map.md)
6. วง pain point ที่เด่นที่สุด 3 จุด โดยยังไม่เสนอเครื่องมือหรือ solution สำหรับการแก้ปัญหา

## Checkpoint

- มีงานอย่างน้อย 8 งาน ครบมากกว่า 1 ระดับ
- pain point ทุกจุดบอกได้ว่าเกิดตรงไหนและสร้างผลกระทบอะไร
- หากคำอธิบายเป็นเพียง “งานเยอะ” ให้ลองถามกันต่อว่า “งานส่วนใด รออะไร และใครได้รับผล” (ไม่งั้นทีม TA กับพลจะเข้าไปถามเองน้า)

## สิ่งที่จะได้จาก exercise นี้

Work Friction Map ที่แยกงานตามระดับและชี้ 3 จุดติดขัดด้วยหลักฐานจากประสบการณ์จริง
