# Exercise: Prioritize the Opportunity

## Exercise Overview

- **เวลา:** 10:45–12:00 (75 นาที)
- **เป้าหมาย:** เปลี่ยน pain point หลายรายการให้เป็น opportunity เดียวที่ชัดและทดลองได้
- **ผลลัพธ์:** กลุ่ม pain-point themes ผลโหวต และ opportunity statement หนึ่งประโยค

## Prerequisites

- Work Friction Map จาก Section 1
- post-it ปากกา dot sticker คนละ 3 จุด flipchart และ [Opportunity Priority Matrix](../../templates/opportunity-priority-matrix.md)

## Scenario 1: เลือก paint point ที่คุ้มค่าลงแรง

ทีมไม่มีเวลาปรับ หรือแก้ทุกปัญหาพร้อมกัน จึงต้องรวมปัญหาที่คล้ายกันและเลือกที่พบบ่อยที่สุด และสร้างความลำบาก และมีผลต่องานจริง

### Practice 1: จัดกลุ่ม โหวต และเขียนโจทย์

#### Steps

1. แต่ละคนเขียนความลำบากจากงานจริง 3 เรื่อง เรื่องละหนึ่ง post-it หรือถ้ามี pain point ที่เขียนไว้แล้วให้เริ่มจากจุดนั้นได้
2. ตั้งชื่อ theme แบบเป็นกลาง เช่น **Delay**, **Rework**, **Communication**, หรือ **Information Overload**
3. ให้คะแนนแต่ละ theme ใน [Opportunity Priority Matrix](../../templates/opportunity-priority-matrix.md) จาก 1–3 ด้าน frequency, frustration, work impact และ feasibility
4. ให้สมาชิกใช้ dot sticker คนละ 3 จุด โหวตได้ทั้งหมดในเรื่องเดียวหรือแบ่งหลายเรื่อง
5. เลือก theme ที่ได้คะแนนสูงและทีมมีอำนาจทดลองได้
6. เขียนประโยคโดยใช้รูปแบบนี้:

> How might we improve **[งาน/ขั้นตอน]** for **[ผู้ใช้]** so that **[ผลลัพธ์]** while still **[ข้อควบคุมสำคัญ]**?

8. อ่านประโยคให้คู่อื่นฟัง หากอีกคู่ยังถามว่า “ปัญหาคืออะไร” หรือ “ใครได้ประโยชน์” ให้แก้ให้เฉพาะเจาะจงขึ้น

## Checkpoint

- opportunity statement ระบุผู้ใช้ งาน ผลลัพธ์ แ
- scope เล็กพอสำหรับการทดลองหนึ่ง workflow
- ประโยคไม่ล็อกคำตอบว่า “ต้องใช้ AI” ก่อนออกแบบงาน

## Expected Output

Opportunity statement หนึ่งประโยคพร้อมหลักฐานสั้น ๆ ว่าทำไมทีมเลือกเรื่องนี้
