# Exercise: Redesign the Human + AI Workflow

## Exercise Overview

- **เวลา:** 13:00–14:30 (90 นาที)
- **เป้าหมาย:** แยก workflow เป็นขั้นตอนและกำหนดบทบาท Human, AI หรือ Shared
- **ผลลัพธ์:** current/future workflow และ AI Use Canvas ฉบับร่าง

## Prerequisites

- Opportunity statement จาก Section 2
- flipchart ปากกา post-it 3 สี
- [Human + AI Workflow Map](../../templates/human-ai-workflow-map.md) และ [AI Use Canvas](../../templates/ai-use-canvas.md)

## Scenario 1: จัด handoff ใหม่ก่อนเพิ่ม AI

ทีมต้องการลดเวลารอ แต่ยังต้องรักษาความถูกต้องและผู้รับผิดชอบ การออกแบบจึงต้องบอกว่า AI ช่วยเตรียมอะไร มนุษย์ตัดสินอะไร และจุดใดต้องทำร่วมกัน

### Practice 1: วาด current state

#### Steps

1. วาง post-it จากซ้ายไปขวา หนึ่งใบต่อหนึ่งขั้นตอน ตั้งแต่ trigger ถึงผลลัพธ์สุดท้าย
2. เขียนผู้รับผิดชอบ input และ output บนแต่ละใบ
3. ทำเครื่องหมายจุดรอ จุดย้อนกลับ และจุดตรวจพวกเราภาพ
4. ถามว่า “ถ้าตัดขั้นตอนนี้ออก งานยังปลอดภัยและมีพวกเราภาพหรือไม่” แล้วตัดสิ่งที่ไม่จำเป็น

### Practice 2: ออกแบบ future state และ AI Use Canvas

#### Steps

1. วาด future workflow ให้สั้นหรือชัดขึ้น โดยไม่ต้องใช้ AI ทุกขั้น
2. ติดป้าย **Human** เมื่อขั้นตอนต้องใช้ judgment, accountability หรือความสัมพันธ์
3. ติดป้าย **AI** เมื่อเป็นการค้นหา สรุป จัดรูปแบบ หรือร่างจากข้อมูลที่อนุญาต
4. ติดป้าย **Shared** เมื่อ AI เตรียมร่างและ Human ตรวจ แก้ หรือตัดสินใจ
5. ระบุ human checkpoint ก่อนการส่ง การอนุมัติ หรือการตัดสินใจที่มีผล
6. ถ่ายข้อมูลจาก workflow ลง [AI Use Canvas](../../templates/ai-use-canvas.md): user, problem, workflow, AI role, data, guardrail และ expected value
7. แลก canvas กับอีกทีมและถามว่า “AI ได้ข้อมูลจากไหน” กับ “ใครตรวจผลลัพธ์”

## Checkpoint

- current state และ future state มีจุดเริ่ม/จบชัดเจน
- ทุกขั้นมี owner และ human checkpoint สำหรับงานสำคัญ
- AI role เป็นงานที่สังเกตได้ เช่น “สรุปอีเมล 10 ฉบับเป็นประเด็น” ไม่ใช่ “ช่วยงานทั้งหมด”

## Expected Output

Human + AI Workflow Map ที่ออกแบบใหม่และ AI Use Canvas ฉบับร่างหนึ่งแผ่น

## Optional Extension

เพิ่ม fallback path สำหรับกรณีข้อมูลไม่ครบ AI ตอบไม่ได้ หรือผู้ใช้ไม่อนุมัติร่าง
