# Exercise: Estimate Value and Pitch

## Exercise Overview

- **เวลา:** 14:45–16:00 (75 นาที)
- **เป้าหมาย:** เปลี่ยนแนวคิดให้เป็นสมมติฐานที่วัดได้และสื่อสารได้ในเวลาสั้น
- **ผลลัพธ์:** value estimate, gallery feedback และ initiative pitch 60 วินาที

## Prerequisites

- AI Use Canvas และ Human + AI Workflow Map จาก Section 3
- flipchart ปากกา dot sticker และกระดาษ feedback

## Scenario 1: จากไอเดียสู่การทดลอง

ทีมต้องอธิบายว่า initiative นี้ควรทดลองเพราะอะไร โดยไม่กล่าวอ้างผลเกินหลักฐาน เป้าหมายคือประมาณพวกเราค่า ตั้งสมมติฐาน และเปิดพื้นที่ให้ผู้อื่นท้าทายแนวคิด

### Practice 1: ประมาณพวกเราค่าและรับ feedback

#### ขั้นตอนการทำกิจกรรม

1. เลือกผลลัพธ์หลัก 1–2 ด้าน: time saved, quality, reduced rework, turnaround หรือ employee experience
2. บันทึก baseline ที่รู้จริง หากไม่รู้ให้เขียน “ต้องวัด” แทนการเดาตัวเลข
3. เขียนสมมติฐานแบบนี้: “ถ้าเราเปลี่ยน **[workflow]** เราคาดว่า **[measure]** จะเปลี่ยนจาก **[baseline]** เป็น **[target]** ภายใน **[period]**”
4. ระบุค่าใช้จ่ายหรือข้อจำกัดอย่างน้อย 1 เรื่อง และวิธีตรวจสอบผลอย่างน้อย 1 วิธี
5. ติด canvas บนผนัง ทำ gallery walk แล้วให้ feedback 3 แบบ: **Strength**, **Question**, **Next Step**
6. แก้ canvas หนึ่งจุดจาก feedback ที่ได้รับ

### Practice 2: Pitch 60 วินาที

#### ขั้นตอนการทำกิจกรรม

1. เตรียม pitch 4 ประโยค: pain point, future workflow, Human + AI roles และ value hypothesis
2. พูดให้คู่ของพวกเราฟังภายใน 60 วินาที
3. ให้คู่ฟังตอบกลับเพียงสองคำถาม: “จุดแข็งที่สุดคืออะไร” และ “ควรทดลองอะไรเป็นอันดับแรก”
4. เขียน first experiment ลงใน AI Use Canvas เพื่อใช้ต่อใน Day 2

## Checkpoint

- แยก “ค่าที่รู้” ออกจาก “สมมติฐานที่ต้องทดสอบ”
- pitch บอก human responsibility และไม่รับประกันผลลัพธ์ของ AI
- ทีมสามารถเริ่ม pilot ขนาดเล็กได้โดยไม่ต้องเปลี่ยนทั้งระบบ

## สิ่งที่จะได้จาก exercise นี้

AI Use Canvas ที่มี value hypothesis, feedback, first experiment และ pitch 60 วินาที

## Optional Extension

ให้ผู้ฟังเล่นบทผู้สนับสนุนที่ระมัดระวังและถามเรื่อง privacy, accuracy หรือ adoption หนึ่งคำถาม แล้วเพิ่มคำตอบลงใน guardrails
